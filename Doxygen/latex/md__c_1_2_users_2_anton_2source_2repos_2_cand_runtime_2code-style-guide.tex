The code committed to this project must follow the Google C++ Style Guidelines unless the guideline conflicts with a C\& Core Project Guideline described below. In this case, the C\& Core Project Guideline will override the Google C++ Style Guideline. In case of any other conflicts, prefer to choose the easy-\/to-\/understand and clear solution which maintains functionality.

\begin{DoxyItemize}
\item Place the most common public interface methods and members at the top of the class definition.
\item Prefer to order methods and variables separately into logical groups closest to call site.
\item Place all private members and methods at the bottom of the class definition.
\item Define methods out-\/of-\/line if the class interface spans over a screen\textquotesingle{}s height, otherwise prefer to define methods inline.
\item Use \textquotesingle{}private\+:\textquotesingle{} and \textquotesingle{}public\+:\textquotesingle{} to separate methods into of classes with large interfaces into logical groups.
\item Postfix virtual base classes\textquotesingle{} name with \textquotesingle{}Base\textquotesingle{} 
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_anton_2source_2repos_2_cand_runtime_2code-style-guide_autotoc_md4}{}\doxysection{\texorpdfstring{Using the standard template library}{Using the standard template library}}\label{md__c_1_2_users_2_anton_2source_2repos_2_cand_runtime_2code-style-guide_autotoc_md4}
When any std library is required include "{}cppsextended.\+h"{} which includes all the standard library headers which this project uses. The specific includes can be found in "{}import\+\_\+stl.\+h"{}. If a required include is not present add it to "{}import\+\_\+stl.\+h"{}. Once C++23 is fully supported by the major compilers, will use "{}import std;"{} instead.
